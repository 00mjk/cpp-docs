# /Zc (Conformance)
You can use the **/Zc** compiler option to specify standard or Microsoft-specific compiler behavior. Here are the **/Zc** compiler options:  
  
-   [/Zc:forScope](../buildref/-zc-forscope--force-conformance-in-for-loop-scope-.md)  
  
-   [/Zc:wchar_t](../buildref/-zc-wchar_t--wchar_t-is-native-type-.md)  
  
-   [/Zc:auto](../buildref/-zc-auto--deduce-variable-type-.md)  
  
-   [/Zc:trigraphs](../buildref/-zc-trigraphs--trigraphs-substitution-.md)  
  
-   [/Zc:rvalueCast](../buildref/-zc-rvaluecast--enforce-type-conversion-rules-.md)  
  
-   [/Zc:strictStrings](../buildref/-zc-strictstrings--disable-string-literal-type-conversion-.md)  
  
-   [/Zc:inline](../buildref/-zc-inline--remove-unreferenced-comdat-.md)  
  
-   [/Zc:implicitNoexcept](../buildref/-zc-implicitnoexcept--implicit-exception-specifiers-.md)  
  
 For more information about conformance issues in Visual C++, see [Nonstandard Behavior](../cpp/nonstandard-behavior.md).
