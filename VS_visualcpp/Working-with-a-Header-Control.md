# Working with a Header Control
The easy way to use a header control ([CHeaderCtrl](../VS_visualcpp/CHeaderCtrl-Class.md)) is in conjunction with a list control; see [Using CListCtrl](../VS_visualcpp/Using-CListCtrl.md) later in this topic family. You can also use a header control by itself. MFC calls **InitCommonControls** for you. The key tasks are as follows:  
  
-   [Creating the header control](../VS_visualcpp/Creating-the-Header-Control.md)  
  
-   [Adding items to the header control](../VS_visualcpp/Adding-Items-to-the-Header-Control.md)  
  
-   [Ordering items in the header control](../VS_visualcpp/Ordering-Items-in-the-Header-Control.md)  
  
-   [Processing header-control notifications](../VS_visualcpp/Processing-Header-Control-Notifications.md)  
  
 If the header control object is embedded in a parent view or dialog class, the control is destroyed when the parent is destroyed.
