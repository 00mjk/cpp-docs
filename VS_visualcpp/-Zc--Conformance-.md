# -Zc (Conformance)
You can use the **/Zc** compiler option to specify standard or Microsoft-specific compiler behavior. Here are the **/Zc** compiler options:  
  
-   [/Zc:forScope](../VS_visualcpp/-Zc-forScope--Force-Conformance-in-for-Loop-Scope-.md)  
  
-   [/Zc:wchar_t](../VS_visualcpp/-Zc-wchar_t--wchar_t-Is-Native-Type-.md)  
  
-   [/Zc:auto](../VS_visualcpp/-Zc-auto--Deduce-Variable-Type-.md)  
  
-   [/Zc:trigraphs](../VS_visualcpp/-Zc-trigraphs--Trigraphs-Substitution-.md)  
  
-   [/Zc:rvalueCast](../VS_visualcpp/-Zc-rvalueCast--Enforce-type-conversion-rules-.md)  
  
-   [/Zc:strictStrings](../VS_visualcpp/-Zc-strictStrings--Disable-string-literal-type-conversion-.md)  
  
-   [/Zc:inline](../VS_visualcpp/-Zc-inline--Remove-unreferenced-COMDAT-.md)  
  
-   [/Zc:implicitNoexcept](../VS_visualcpp/-Zc-implicitNoexcept--Implicit-Exception-Specifiers-.md)  
  
 For more information about conformance issues in Visual C++, see [Nonstandard Behavior](../VS_visualcpp/Nonstandard-Behavior.md).
